This module helps integrate Odoo HR Attendance with ZK attendance machines, enabling seamless synchronization of attendance records between the devices and Odoo. In addition, this module supports the ADMS protocol, allowing dynamic communication with attendance devices without requiring static IP configurations.



Cloud/ADMS protocol

A new feature called ADMS was introduced to enhance device-to-software communication. Traditionally, the system relied on obtaining data from devices using static IPs and modem configurations, With the ADMS feature, devices can now push data directly to the software, eliminating the need for static IP and modem setups.

The ADMS feature enables real-time or scheduled data transmission through HTTP protocols, allowing devices to actively communicate with the server. In this setup, the device pushes its data to the server, while any server-side commands are returned when the device periodically checks for updates. This new approach simplifies the configuration process, reducing potential technical challenges for clients.

Configure the device with Odoo (Attendances >> Biometric Device Config)

Configure a static IP, Port and Timezone for the system. If the biometric device has password, you can configure it by enabling 'Is Password Set' field. After configuration click the 'Test Connection' button.

If your connection success, server shows a success message

If your connection fails, server shows an error message

Configure the device with Odoo (Attendances >> Biometric Device Config) (IF ADMS Enabled)

Once the serial number is set and ADMS is enabled, you will need to configure your server's IP address on the biometric device.

Follow these step-by-step instructions for the configuration.

Device Configuration
  1. Access Machine Settings:
    • Navigate to the 'Comm' setting of the machine.
  2. Configure Cloud Server Settings:
    • Enable Domain Name: Activate this if you plan to use a domain.
    • Server Address: Enter the IP (e.g., 192.168.1.1) or domain (e.g., example.com).
    • You can find your IP in the internet settings.
    • Server Port: Use 443 for online servers or the port your server uses (e.g., 8069 for local Odoo environments).
    • Enable Proxy Server (if available): Activate it and provide the proxy server’s IP and port.
    • HTTPS (if available): Enable HTTPS for secure data transfer unless using a local environment.
For further reference: Biotime 8 – Communication Settings

Once the device configuration is complete, it will send a request to the server with its serial number. At that point, the Odoo server will verify if the device exists by checking the serial number. If the device is found, its status will automatically be updated to "Connected."

After completing the device configuration, if your device fails to connect to the Odoo server, it will remain in the 'Not Connected' status. You only need to wait for a while (about 1 minute) for the connection to establish.

Synchronize employees with biometric device

Click the 'Sync Employees' button to synchronize employees with biometric device.

If employees synced with biometric device, you can see a popup with success message.

You can see three menus in employee form.
1. Scan with biometric - Click to scan employee with biometric device
2. Update with biometric - Click to update employee details with biometric device
3. Remove from biometric - Click to remove employee details from biometric device

Click Scan to enroll employee with biometric device.

Click 'Update' to update employee details with biometric device.

Click 'Remove 'to remove employee details from biometric device.

After updating the employee details, a line will create under the 'Biometric Devices' tab. This line maps the same ID present on the biometric device to the Biometric User Id field.

Synchronize employees with biometric device (IF ADMS Enabled)

Click the 'Export Employees' button to synchronize employees with biometric device.

When the 'Export Employee' button is clicked, device commands are created to register users on the device. Once the command is successfully executed (which takes about 30 seconds), the employees will be synced with the device users.

Download Attendance Logs

Click the 'Download Attendance' button to download attendance log.

After downloading attendance log server shows a success message.

Under the Manager menu, you can see three new menus.

1. Biometric Attendance Log - shows the downloaded attendance log of employees
2. Calculate Attendance - calculate the attendance and shows the difference in attendances menu.
3. Attendance Report - excel report of attendance
